# ADR-004: PUUID для внутренней логики

**Статус:** Принято  
**Дата:** 2026-05

## Контекст

Riot ID (Имя#Тег) может меняться при переименовании; навигация и избранное требуют стабильного ключа.

## Рассмотренные альтернативы

| Вариант | Плюсы | Минусы | Решение |
|---------|-------|--------|---------|
| **PUUID** | Постоянный ID Riot | Не читаем пользователем | ✅ Внутренний ключ |
| Riot ID в URL | Читаемость | Ломается при rename | Только для поиска |
| summonerId (legacy) | Числовой ID | Deprecated Riot API | Отклонено |
| Имя + регион | Простота | Дубликаты, смена ника | Отклонено |

## Решение

- **Поиск в UI:** Riot ID (`Имя#Тег`)
- **Избранное, навигация, матчи:** PUUID
- API: `POST /summoner/favorites` принимает `{ "puuid": "..." }`
- Mobile: `navigation.navigate('Profile', { puuid, region })`

## Последствия

- (+) Стабильные ссылки при смене отображаемого имени
- (−) PUUID не отображается пользователю напрямую
