# ADR-001: Выбор архитектурного паттерна PCMEF

**Статус:** Принято  
**Дата:** 2026-04

## Контекст

Курсовой проект требует слоевой архитектуры с чётким разделением ответственности. Траектория В предполагает мобильный клиент и Java backend.

## Рассмотренные альтернативы

| Вариант | Плюсы | Минусы | Решение |
|---------|-------|--------|---------|
| **PCMEF** | Соответствие методичке; тестируемость Mediator | Два репозитория (mobile + backend) | ✅ Выбрано |
| MVC monolith (Thymeleaf) | Один деплой | Не мобильная траектория | Отклонено |
| Clean Architecture (Uncle Bob) | Гибкость | Избыточно для учебного объёма | Отклонено |
| Микросервисы | Масштабирование | Сложность для 16 эндпоинтов | Отклонено |

## Решение

Enterprise-адаптация PCMEF:
- Presentation — React Native (Expo)
- Control, Mediator, Entity, Foundation — Spring Boot
- Интеграция через REST/JSON + JWT

## Последствия

- (+) Единая бизнес-логика на сервере
- (+) Клиент остаётся тонким
- (−) Сетевая зависимость mobile от backend
