# Бизнес-глоссарий предметной области

**Проект:** мобильное приложение-компаньон для League of Legends  
**Версия документа:** 1.0  
**Назначение:** единая терминология для аналитики, разработки и курсовой документации (PCMEF, Spring Boot, React Native, Riot Games API).

| № | Термин | Определение | Примечание в контексте системы |
|---|--------|-------------|------------------------------|
| 1 | **Призыватель (Summoner)** | Игрок LoL, имеющий учётную запись Riot с уникальным игровым именем (ник), уровнем аккаунта и историей матчей. | Основной объект поиска и отображения в приложении; хранится в сущности `Summoner`. |
| 2 | **Игровое имя (Summoner Name / Riot ID)** | Отображаемый ник призывателя в выбранном регионе; может меняться, в отличие от PUUID. | Вводится пользователем на экране поиска; передаётся в запрос `GET /summoner/{name}`. |
| 3 | **PUUID** | Постоянный универсальный идентификатор игрока в экосистеме Riot; не меняется при смене ника. | Ключ для запроса истории матчей и связи записей в БД; поле `puuid` в `Summoner`. |
| 4 | **Регион (Platform / Routing Region)** | Игровой сервер (например, `EUW1`, `RU`, `NA1`), в рамках которого действует аккаунт и API-маршрутизация. | Задаётся в конфигурации бэкенда (`riot.api.default-region`); влияет на поиск и лиги. |
| 5 | **Матч (Match)** | Одна завершённая игровая сессия (классически 5×5) с уникальным `matchId`, режимом, длительностью и статистикой участников. | Сущность `Match`; детали подгружаются через Match-V5 API и кэшируются в PostgreSQL. |
| 6 | **Match ID** | Строковый идентификатор матча в Riot API (например, `EUW1_7123456789`). | Первичный бизнес-ключ матча в локальной БД. |
| 7 | **KDA** | Показатель боевой эффективности: убийства (K), смерти (D), ассисты (A); часто вычисляется коэффициент `(K + A) / max(D, 1)`. | Отображается в карточке матча и аналитике чемпионов. |
| 8 | **CS (Creep Score)** | Количество последних ударов по миньонам и нейтральным юнитам; мера фарма на линии/в лесу. | Поле `csScore` в статистике участника матча. |
| 9 | **MMR (Matchmaking Rating)** | Скрытый рейтинг подбора соперников; напрямую не отдаётся Riot API, но влияет на качество и скорость матчмейкинга. | В приложении может упоминаться в пояснениях к рангу; не хранится как отдельное поле API. |
| 10 | **Ранг (Rank / Tier)** | Видимая ступень соревновательного рейтинга: Iron … Challenger (включая Master+ без подрангов). | Поле `tier` в профиле; для Unranked — отдельное состояние. |
| 11 | **Подранг (Rank Division)** | Деление внутри тира: I, II, III, IV (кроме Master+). | Поле `rank` (например, `I`, `II`). |
| 12 | **LP (League Points)** | Очки в рамках текущего дивизиона; накопление ~100 LP ведёт к серии повышения. | Поле `leaguePoints` в DTO профиля. |
| 13 | **Винрейт (Win Rate)** | Доля побед в ранговых играх за сезон, в процентах. | Агрегируется бэкендом и отдаётся клиенту (`winRate`). |
| 14 | **Чемпион (Champion)** | Играбельный персонаж с уникальным набором способностей и ролью на карте. | Идентификатор/имя в `ParticipantStats`, агрегация в `ChampionStats`. |
| 15 | **Роль (Role / Position)** | Позиция на карте: Top, Jungle, Mid, ADC (Bot), Support. | Используется при аналитике пула чемпионов и состава матча. |
| 16 | **Режим игры (Game Mode)** | Тип очереди: `RANKED_SOLO_5x5`, нормальные, ARAM и др. | Поле `gameMode` в сущности матча. |
| 17 | **Участник матча (Participant)** | Запись об игроке внутри конкретного матча (чемпион, KDA, золото, победа/поражение). | Сущность `ParticipantStats`, связана с `Match`. |
| 18 | **Драфт (Champion Select)** | Фаза выбора и бана чемпионов до старта матча. | В API отражается косвенно (баны/пики в деталях матча при расширении функционала). |
| 19 | **Бан (Ban)** | Исключение чемпиона из пула выбора в текущей партии. | Термин предметной области LoL; опционально в детализации матча. |
| 20 | **Riot Games API** | Официальный набор REST-сервисов Riot для данных аккаунтов, матчей и лиг. | Интеграция в слое Mediator (`RiotApiClient`); требует `RIOT_API_KEY`. |
| 21 | **Rate Limit (лимит запросов)** | Ограничение частоты обращений к API (заголовки `X-App-Rate-Limit`, код `429`). | Обрабатывается `RiotRateLimitException` и сценариями ошибок UC-001. |
| 22 | **Кэш профиля (Cache-Aside, TTL)** | Стратегия: сначала чтение из PostgreSQL; при устаревании — обновление из Riot API. | TTL профиля призывателя — 10 минут (`Summoner.isCacheFresh()`). |
| 23 | **Избранное (Favorites)** | Список призывателей, сохранённых пользователем для быстрого доступа без повторного поиска. | Связь `User` ↔ `Summoner` через `UserFavoriteSummoner`; дублирование в `AsyncStorage` на клиенте. |
| 24 | **Пользователь приложения (User)** | Владелец локальной/серверной учётной записи компаньона (не путать с призывателем LoL). | Сущность `User`; владеет списком избранных. |
| 25 | **Мобильный компаньон** | Клиентское приложение (React Native): поиск, профиль, матчи, избранное. | Слой Presentation; обмен с бэкендом через Axios. |
| 26 | **PCMEF** | Архитектурный стиль проекта: Presentation → Control → Mediator → Entity → Foundation. | Контроллеры не вызывают Riot API напрямую; интеграция только в Mediator. |

## Сокращения

| Сокращение | Расшифровка |
|------------|-------------|
| API | Application Programming Interface |
