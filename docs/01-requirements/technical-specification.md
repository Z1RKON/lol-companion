## 1. Общие сведения

### 1.1 Полное наименование системы

Автоматизированная информационная система «LoL Companion».

### 1.2 Краткое наименование

LoL Companion.

### 1.3 Основание для разработки

Методические указания по выполнению курсового проекта «Программная инженерия», СКФУ, 2026.

## 2. Назначение и цели

### 2.1 Назначение

Предоставление игрокам League of Legends мобильного доступа к статистике призывателей (ранг, винрейт, история матчей) с персональным списком избранных игроков.

### 2.2 Цели

1. Реализовать клиент-серверную архитектуру PCMEF.
2. Интегрировать Riot Games API с кэшированием в PostgreSQL.
3. Обеспечить JWT-аутентификацию и ≥ 8 REST-эндпоинтов.
4. Достичь покрытия автотестами ≥ 40% (backend и mobile).

## 3. Характеристика объекта автоматизации

Игроки League of Legends, использующие внешние сайты (op.gg, u.gg) для просмотра статистики. Требуется единое мобильное приложение с офлайн-кэшем избранного.

## 4. Требования к системе

### 4.1 Функциональные требования

| ID | Требование |
|----|------------|
| FR-01 | Регистрация и вход пользователя |
| FR-02 | Поиск призывателя по Riot ID и региону |
| FR-03 | Просмотр профиля: ранг, LP, винрейт |
| FR-04 | История матчей и детали матча |
| FR-05 | Избранное: добавление, просмотр, удаление |
| FR-06 | Привязка собственного Riot ID к аккаунту |
| FR-07 | Кэширование данных Riot API (TTL 10 мин) |

### 4.2 Нефункциональные требования

| ID | Требование |
|----|------------|
| NFR-01 | Backend: Java 17, Spring Boot 3.3 |
| NFR-02 | Mobile: React Native (Expo), TypeScript |
| NFR-03 | СУБД: PostgreSQL 14+ |
| NFR-04 | Покрытие тестами ≥ 40% |
| NFR-05 | Документация по этапам 0–12 в `docs/` |
